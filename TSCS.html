How to use the simulation

  1. Select the number of processes that you want to be present in the simulation.
  2. Select the speed of the simulation, this can be changed after starting the animation but you have to click 'Pause/Resume Simulation' after changing from step to fast or slow.
  3. To pause the simulation with fast or slow speeds click on 'Pause/Resume Simulation'.
  4. To reset the simulation, click on 'Reset Simulation'.

The Timestamp Based Critical Section Algorithm

The purpose of this algorithm is to ensure that distributed processes do not access a shared resource (critical section) at the same time. This is accomplished by making use of Lamport Timestamps to assign every request a clock value. Before a process enters the critical section it waits on acknowledgements from every other process. Once finished, it lets all the other processes know that it is done and they each remove that request from their queues, then the next process enters.

Pseudocode to request the critical section for process i:
 queue.push(i, timestamp)
 broadcast_request(i, timestamp)

Pseudocode to enter the critical section for process i:
 queue.sort
 if queue[0] == i and i.acks == number_of_processes - 1 do
   critical_section.enter()
 end if
